# 通用领域 AI 学习助手

## 项目定位

本项目计划构建一个通用领域的 Web AI 学习助手，帮助用户围绕任意学习主题上传资料、梳理目标、生成学习路径，并通过问答、分层讲解、阶段测验和反馈总结完成持续学习。

项目不限定于金融、编程、医学、法律、考试、语言学习等固定方向，而是以“用户上传资料 + 用户自定义学习目标”为核心，让系统根据资料内容和用户基础动态组织学习流程。

## 核心目标

- 支持用户上传不同领域的学习资料。
- 根据用户的学习目标、当前水平和可用时间生成学习计划。
- 基于资料内容进行检索增强问答，减少脱离资料的泛泛回答。
- 将复杂内容拆解为由浅入深的讲解。
- 生成阶段测验，帮助用户检查掌握情况。
- 记录学习会话、学习路径、问答历史、测验结果和反馈。
- 作为一个 AI 应用开发展示项目，体现文档处理、向量检索、Agent 编排、LLM 调用和前端交互的完整流程。

## 计划技术栈

- 后端：FastAPI
- 前端：FastAPI 模板 + 原生 JavaScript
- 数据库：SQLite
- Agent 编排：LangGraph
- 向量库与检索：
  - Chroma：持久化文档、文本块和向量元数据
  - FAISS：本地相似度检索能力展示
- LLM：OpenAI 兼容接口
- 文件类型：PDF、Markdown、TXT

## 主要功能模块

### 1. 资料接入

用户可以上传不同领域的学习资料。系统会完成文本抽取、文本切分、向量化和索引入库。

计划支持：

- PDF
- Markdown
- TXT

### 2. 学习会话

用户可以创建一个学习会话，并填写：

- 学习主题
- 当前基础
- 学习目标
- 可用学习时间
- 希望的学习风格或深度

系统根据这些信息生成个性化学习路径。

### 3. 学习路径规划

系统根据用户目标和资料内容生成阶段化学习路径，例如：

- 入门概念
- 核心知识
- 关键方法
- 应用练习
- 复习与测验

具体阶段不写死，由资料内容和用户目标动态生成。

### 4. 分层讲解

同一知识点可以按不同层级讲解：

- 简明版
- 详细版
- 示例版
- 面向初学者版
- 面向进阶学习者版

### 5. 问答辅导

用户可以围绕当前学习阶段提问。系统优先基于已上传资料检索相关片段，再结合 LLM 生成回答，并展示参考来源。

### 6. 阶段测验

系统可以根据当前学习阶段生成测验题，帮助用户检查理解情况。

测验结果会被记录，并用于生成学习反馈。

### 7. 学习反馈

根据用户问答历史和测验表现，系统生成反馈，包括：

- 已掌握内容
- 薄弱知识点
- 建议复习方向
- 下一步学习建议

## 计划 API

- `POST /api/documents/upload`
  上传学习资料并建立索引。

- `GET /api/documents`
  查看已上传资料和索引状态。

- `POST /api/learning/sessions`
  创建学习会话。

- `GET /api/learning/sessions/{id}`
  获取学习会话详情、学习路径、历史问答和测验结果。

- `POST /api/learning/sessions/{id}/plan`
  生成学习路径。

- `POST /api/learning/sessions/{id}/chat`
  基于当前学习阶段和资料内容进行问答。

- `POST /api/learning/sessions/{id}/quiz`
  生成阶段测验。

- `POST /api/learning/sessions/{id}/quiz/submit`
  提交测验答案并生成反馈。

## AI 应用流程

```text
上传资料
  -> 文本抽取
  -> 文本切分
  -> 向量化
  -> Chroma / FAISS 索引
  -> 创建学习会话
  -> LangGraph 编排学习流程
  -> 检索增强生成
  -> 学习路径 / 问答 / 测验 / 反馈
  -> 保存学习状态
```

## 环境变量规划

```text
OPENAI_API_KEY=你的 API Key
OPENAI_BASE_URL=OpenAI 兼容接口地址
OPENAI_MODEL=使用的模型名称
```

代码中不写死具体模型厂商，只通过 OpenAI 兼容接口配置。

## 已确认需求

1. 用户体系采用多用户模式，但只做简单实现。
2. 需要注册和登录功能，并记录不同用户的账号和密码。
3. 注册字段只需要账号和密码。
4. 知识库属于用户个人信息，不同用户之间保持独立，不能相互查看。
5. 一个用户可以创建多个知识库。
6. 上传资料按不同知识库分组，不需要更复杂的学科或项目层级。
7. 学习会话不强制用户手动绑定单个知识库，允许 Agent 或 LLM 根据用户问题和学习目标智能选择合适知识库。
8. Agent 选择知识库时，只选择最相关的一个知识库。
9. 学习会话不需要额外保存“本次使用过哪些知识库”。
10. 学习路径允许用户手动编辑，包括调整阶段、删除阶段和新增阶段。
11. Agent 可以根据用户学习情况、问答记录、测验准确率和反馈动态调整学习路径。
12. Agent 动态调整学习路径后，直接覆盖当前路径，不保留历史版本。
13. 学习路径中的阶段需要显示学习状态，例如未开始、学习中、已完成。
14. 阶段测验题型以选择题、判断题、简答题为主。
15. 测验不需要保存每道题的完整批改明细，但需要保存足够信息，让 Agent 根据准确率理解用户学习情况。
16. 上传资料需要限制文件大小、数量或容量，具体限制可由实现阶段自行决定。
17. 允许用户删除知识库和资料。
18. 删除知识库时，同步删除该知识库下的所有资料、向量索引、学习进度和相关信息。
19. 对话窗口需要支持流式输出。
20. 对话内容需要支持 Markdown 渲染。
21. API 调用信息后续由用户提供，当前只预留 OpenAI 兼容接口配置。
22. 无 API Key 的 mock 演示模式不是当前重点，项目代码搭建后由用户提供真实 API 调用信息。
23. 前端偏学习平台风格，需要对话窗口，并预留知识地图、用户信息、不同知识库管理等窗口空间。
24. 知识地图第一版只预留区域，不要求生成节点关系图。
25. 用户信息页展示账号信息、历史会话和学习进度。
26. 知识库信息单独放在一个页面中管理。
27. 点击某个知识库后，可以查看该知识库下的学习进度、上传资料、知识路径、知识点提取和用户学习情况。
28. 不需要管理员角色，所有用户权限相同。
29. Django 可作为未来管理后台扩展方向保留在架构说明中。

## 后续待确认需求

后续实现前，需要继续确认以下问题：

1. 知识库字段是否只需要名称和简介，还是还需要标签、封面、创建时间、更新时间？
2. Agent 的学习流程是否固定为“诊断 -> 选库 -> 检索 -> 规划 -> 讲解 -> 问答 -> 测验 -> 反馈 -> 调整路径”？
3. 用户创建学习会话时，需要填写哪些信息：学习主题、当前水平、学习目标、可用时间、偏好风格是否都需要？
4. Agent 选择知识库的依据是什么：知识库名称、简介、已提取知识点、文档摘要、用户历史学习情况，哪些必须参与？
5. 知识点提取需要在上传资料后自动执行，还是点击知识库时再生成？
6. 学习路径的结构需要包含哪些字段：阶段名称、目标、知识点、推荐资料、练习、状态、预计时间？
7. Agent 调整学习路径的触发条件是什么：测验准确率低、用户主动要求、连续问某类问题、阶段完成后自动触发？
8. 检索结果是否需要在回答中展示来源文档和片段摘要？
9. 简答题如何判分：LLM 给出分数即可，还是只做文字反馈？
10. 是否需要记录用户每次对话，用于后续学习状态分析？
